# Правила участия

Проект развивается как рабочий инженерный инструмент. В обсуждениях и изменениях ожидаются уважение, конкретика и ориентация на результат.

## Ожидаемое поведение

- Формулировать проблемы через факты, воспроизводимые шаги и ожидаемый результат.
- Обсуждать код, архитектуру и поведение программы, а не личности.
- Проверять изменения тестами или понятным smoke-сценарием.
- Не публиковать секреты: Telegram token, приватные chat id, `.env`, ключи API.
- Уважать ограничения источников данных и не выдавать проект за торговую рекомендацию.

## Нежелательное поведение

- Оскорбления, травля, дискриминационные высказывания.
- Публикация чужих секретов или персональных данных.
- Намеренное внесение вредоносного кода.
- Маскировка breaking changes под мелкие правки.

## Как сообщать о проблемах

Для ошибок лучше прикладывать:

- команду или Telegram-сообщение;
- диапазон дат;
- тикеры и рынки;
- фрагмент лога без секретов;
- ожидаемый и фактический результат.
